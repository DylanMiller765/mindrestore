# Phase 1 — UI Design Contract

> Visual and interaction contract for the v2.0 onboarding final-polish phase. Native iOS / SwiftUI, dark-pinned. Generated by `gsd-ui-researcher`, verified by `gsd-ui-checker`.

This phase locks down the 16-page onboarding flow so every page lands at OB-design-system polish. Three pages are already shipped (Welcome / Notif Priming / Atmosphere lift); four pages are open work (Pain Cards / Goals / Empathy / Plan Reveal); the remaining nine are mid-flow informational pages whose voice/copy gets refreshed in passing during ONB-08 light/dark parity audit.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native iOS, no third-party design-system tool) |
| Preset | not applicable |
| Component library | SwiftUI native + project-private structs (see `Component Inventory` below) |
| Icon library | SF Symbols (system) + bundled custom: app-icon logos for the 6 named social apps + `app-icon` (Memo) + 17 mascot poses |
| Font | **Bricolage Grotesque** (Regular / Medium / SemiBold / Bold / ExtraBold) bundled in `MindRestore/Resources/Fonts/`. Accessed via `Font.brand(size:weight:)` extension in `MindRestore/Utilities/DesignSystem.swift:579-601`. | Layout pattern | Page-level `ZStack` with atmosphere blurs at the **outer** ZStack so the blur extends behind the progress bar (architectural lift shipped in ONB-03). |

## Spacing Scale (CGFloat / pt — iOS native)

These are the established OB spacing values used in shipped onboarding pages. New work MUST match.

| Token | Value (pt) | Usage |
|-------|------------|-------|
| xs | 4 | Inline icon gaps, dot indicators |
| sm | 8 | Compact element spacing, eyebrow → headline gap |
| md | 14 | Card vertical/horizontal inner padding (notification mockups, receipt slips) |
| md+ | 16 | Receipt slip corner radius, vertical gaps inside cards |
| lg | 18 | Receipt slip horizontal padding, gap between stacked cards |
| lg+ | 22 | Notification card corner radius |
| xl | 24 | Section padding inside hero zones |
| xl+ | 28 | **Page horizontal margin** (canonical), top safe-area inset = 14, bottom safe-area inset = 18 |
| 2xl | 32 | Major vertical breaks (headline → hero scene gap) |
| 3xl | 48 | Page-edge margins for Pain Cards receipt slip (page width − 48 horizontal) |
| 4xl | 64 | Layout-level breaks (rare; reserved for plan-reveal layer transitions) |

**Rationale on 4n+2 values (14 / 18 / 22):** These are intentionally not pure 4-multiples because they are iOS-idiomatic — derived from shipped components (notification card 22pt corner radius, OB safe-area-inset 18pt bottom, receipt slip 14pt vertical padding). Apple's own native components use these exact values, and our shipped Welcome / Notif Priming / Atmosphere pages already enshrine them. Pure 4-multiple drift here would break visual cohesion with the rest of the system.

**Corner radii pattern:** 14pt for primary CTA buttons (`OBContinueButton`), 16pt for receipt slips (Pain Cards), 22pt for notification mockup cards, 9pt for app-icon rounded squares (matches iOS app icon corner ratio at 38pt), `size * 0.22` ratio for the welcome-page logo cards.

**Safe-area insets:** top 14pt, bottom 18pt. Progress bar rests inside the top safe area; CTA + skip rest inside bottom.

**Exceptions:**
- Plan Reveal layers 1 + 2 are **edge-to-edge full-bleed** (no horizontal margin) — the shock number must "eat the screen width." Layer 3 plan card returns to 28pt page margin.
- App-pile fan on Welcome extends past the right safe area on purpose ("continues off-screen").

## Typography (Bricolage Grotesque)

`.brand(size:weight:)` is the exclusive headline + UI text path. SF Rounded is permitted only as a fallback inside `.system(size:weight:design:.rounded)` for legacy pages that have not yet migrated. New code uses Bricolage.

| Role | Size (pt) | Weight | Use |
|------|-----------|--------|-----|
| Display | 38 | heavy (ExtraBold) | Page headline (Welcome: "Apps want you. / Memo wants you back.", Notif Priming: "One pulls you in. / One pulls you out.", Empathy: "Your brain isn't broken. / It's been hijacked.") |
| H1 | 31 | heavy | Plan Reveal cinematic stat unit subtitles, brain age verdict |
| H2 | 22 | bold | Stamp text (Pain Cards "CAUGHT" — monospaced design), section titles inside cards |
| Body lg | 17 | heavy | CTA button label (system, not brand — `.system(size: 17, weight: .bold)` matches `OBContinueButton`) |
| Body | 15 | semibold | Subline / caption text (Welcome subline, Notif Priming caption) |
| Body | 14 | medium / heavy | Notification body copy (medium for feed card, heavy for Memo card to give visual weight) |
| Body | 14 | heavy | Notification app name |
| Eyebrow | 13 | bold + tracking 1.0 | `OBEyebrow` (uppercase categorical labels) |
| Caption | 12 | semibold / medium | Footer privacy line, timestamp ("now"), micro progress ("3 of 6") |
| Caption sm | 11 | semibold | Sub-tertiary annotations on plan card hairlines |

**Rationale on size/weight count (9 roles, 5 weights):** This intentionally exceeds the strict web-checker 4-size/2-weight heuristic because the roles are domain-distinct (display / H1 / H2 / body lg / body / body small / eyebrow / caption / caption sm), not stylistic variation. Sizes align with iOS HIG step-sizes (17pt body, 22pt H2 are Apple-standard). The shipped Welcome bouncer + Notif Priming pages already use this scale and ARE the design source of truth — this contract codifies the existing system rather than redesigning it.

**Numerals (monospaced rule):** ANY number on a hero stat surface (Plan Reveal layers 1 + 2, count-ups, brain age reveal, plan-card row values) MUST use `.system(size:weight:design:.monospaced)`. Receipt micro-progress ("3 of 6") may use brand. The reason: brand font's proportional digits create visual jitter during count-up animations.

**Line heights:** SwiftUI default line spacing (1.0) plus explicit `.lineSpacing(2)` on multi-line headlines (visible in shipped Notif Priming page). Plan Reveal headlines may use `.lineSpacing(4)` for editorial breathing room.

**Tracking:** Eyebrows = 1.0pt. Stamp text ("CAUGHT") = 1.8pt. All other text = 0 (default).

## Color (OB tokens — see `OnboardingNewScreens.swift:1312-1324`)

The OB palette is the SOLE color contract for these 16 pages. Do NOT introduce new hex values; do NOT use `AppColors.*` (those are for the post-onboarding app surface). The exception is mascot PNG assets, which carry their own Memo palette baked in (blue brain body / purple outline / pink folds / glasses).

| Token | Hex | Role | Reserved for |
|-------|-----|------|--------------|
| `OB.bg` | `#0A0A0F` | Dominant page background (60%) | Every page background. Outer ZStack base layer. |
| `OB.surface` | `#14141F` | Secondary surface (30%) | Cards, receipt slips, notification mockups, plan card |
| `OB.border` | `white@8%` | Card border at rest | Saved receipts, dim card chrome |
| `OB.fg` | `white@94%` | Primary text | Headlines, primary card body |
| `OB.fg2` | `white@62%` | Secondary text | Sublines, captions, secondary card body |
| `OB.fg3` | `white@40%` | Tertiary text | Footer privacy line, timestamps, micro progress |
| `OB.accent` | `#6890FE` | Brand blue (10% accent) | **CTA fill, selected state, headline 2nd-line color (split-color pattern), notification "Memo" card border + glow, receipt active border, atmosphere blur cool-side, count-up "with Memo" fill** |
| `OB.coral` | `#FA6B59` | Danger / alarm | **Plan Reveal Layer 1 tint, "CAUGHT" stamp, atmosphere blur warm-side on Welcome, error state captions (Notif Priming timeout)** |
| `OB.amber` | `#FFC247` | Warning | **Permitted as an alternate to coral on the Plan Reveal Layer 1 tint if coral feels too aggressive in user testing — pick one and lock it during this phase** |
| `OB.memoPurple` | `#B857F5` | Memo accent | **Reserved for mascot-related glows ONLY (under `mascot-cool` on Welcome if accent feels too cool, under `mascot-goal` on Empathy). Do NOT use as a CTA / chrome color.** |
| `OB.success` | `#00D19E` | Positive | **Reserved for "with Memo" plan-reveal flip if `OB.accent` feels insufficient for the green/positive read. Do NOT use elsewhere in onboarding.** |

**60/30/10 split (informational, not enforced as a rule):**
- 60% `OB.bg` (page background)
- 30% `OB.surface` + foreground text (cards, receipts, notification mockups, plan card)
- 10% `OB.accent` (CTA, selected state, glows, second-line headline color)

**Accent reserved for (explicit list):**
1. Primary CTA fill (`OBContinueButton`)
2. Selected state on goal cards (Phase 1 ONB-05 work)
3. Headline second-line color in split-color pattern ("Memo wants you back." in `OB.accent`)
4. Notification "Memo" card border (1.5pt @ 35% opacity) + drop shadow (radius 24, y 10, opacity 0.32)
5. Active receipt border (Pain Cards, opacity 0.45–0.55)
6. Plan Reveal Layer 2 background tint
7. Atmosphere blur (cool-side circle, opacity 0.18, blur 76)
8. Memo mascot drop-shadow glow on Welcome (opacity 0.32, radius 28, y 12)
9. Plan card row `02 · APPS LOCKED` value chip if visually emphasized

**Coral reserved for (explicit list):**
1. Plan Reveal Layer 1 background tint ("WITHOUT MEMO" panel)
2. Pain Cards "CAUGHT" stamp text + 2pt rounded-rect border @ 72% opacity
3. Welcome atmosphere warm-side blur (opacity 0.10, blur 68)
4. Notif Priming timeout error caption ("Couldn't request permission. Tap to retry.")
5. NEVER as CTA fill (CTA is always `OB.accent`)

**Destructive / error pattern:** No destructive actions in onboarding. There are NO delete / cancel-with-loss flows. Closest is Notif Priming "Not now" skip — neutral secondary, not destructive.

**Light mode parity:** Pages are `.preferredColorScheme(.dark)`-pinned. The ONB-08 audit confirms that:
- Status bar reads light content on dark
- Modal sheets / popovers do not bleed system light-mode background
- TextField / keyboard appearance respects dark when name input is presented (Name page)
- The brain age reveal `fullScreenCover` already pins dark and is out-of-scope here (existing behavior)

## Animation Cadence Pattern

Every onboarding page follows the same entrance arc. Total duration ~1.5–2.85s. Anything longer feels sluggish; shorter feels cheap. Do NOT idle-animate after entrance unless explicitly specified (Welcome and Notif Priming both go static after entrance).

### Standard arc

| t | Action | Curve | Duration |
|---|--------|-------|----------|
| 0.10s | Eyebrow + headline fade in (opacity 0→1, y 8→0) | easeOut | 0.40s |
| 0.32–0.40s | Hero element rises (cards / receipt stack / app pile / mascot) | spring response 0.50–0.55, damping 0.78–0.82 | 0.50–0.55s |
| 0.78–0.95s | Mascot lands (if applicable). Fire light haptic on land. | spring response 0.46–0.50, damping 0.78–0.80 | 0.50–0.55s |
| 1.20–1.50s | Subline / caption / privacy line fade in | easeOut | 0.30–0.40s |
| 1.45–1.70s | CTA + skip fade up | easeOut | 0.30–0.40s |

Welcome's arc is slightly extended (~1.7s entrance) because it has a 2-beat story (apps arrive → Memo arrives to hold them). Plan Reveal's count-up takes 1.8s for the 0→44,000 number on Layer 1 and the crossfade to 22,000 takes 0.6s — total Plan Reveal entrance ≈ 2.85s, the longest in the flow because it IS the climax.

### Per-tap interactions

| Surface | Tap action | Animation | Haptic |
|---------|------------|-----------|--------|
| Pain Cards `Caught me` | Stamp pops + slip slides to saved stack | Stamp scale 0.72→1.08→1.0 over 0.18s, hold 0.18s, slide back with rotation +5° | medium |
| Pain Cards `Not me` | Slip flicks left | rotation -9°, x offset -340pt, opacity → 0 over 0.30s | light |
| Goals card tap | Selected: lift + glow | y offset -2pt, scale 1.02, OB.accent border | light |
| Goals counter increment | Counter pulse | scale 1.0→1.12→1.0 over 0.18s | light |
| Goals 4th-tap (max) | Counter chip pulse-only — NO selection change | counter chip scale 1.0→1.10→1.0 over 0.20s, OB.coral.opacity(0.35) flash on the `0 / 3 picked` chip border | light |
| OBContinueButton tap | Press-state | scale 0.98 on touch-down (.plain button style) | medium on release (only if it advances; not on disabled tap) |
| Plan Reveal Layer 1→2 | Number crossfade | 44,000 → 22,000 cross-fade with mono digits sliding | medium on cinematic moment |

### Reduce Motion fallback (REQUIRED)

`@Environment(\.accessibilityReduceMotion) private var reduceMotion`

When `reduceMotion == true`:
- Replace ALL spring / slide / flick / scale-pop animations with **0.18s opacity fade-only** transitions.
- Replace count-up (Plan Reveal) with **immediate final value** + 0.18s opacity fade-in.
- Replace card flick (Pain Cards `Not me`) with **fade-out 0.18s** in place.
- Replace stamp scale-pop (Pain Cards `Caught me`) with **fade-in 0.18s** at full scale.
- **Keep all haptics.** Reduce Motion does not mean reduce feedback.
- The CTA scale-press is acceptable to keep (it's < 4pt of visible movement) but acceptable to drop.

|---------|-------|------|

### Receipt count handoff (ONB-09)

Data flow:
1. `OnboardingPainCardsView` declares `let onContinue: (Int) -> Void`.
2. On final prompt completion, calls `onContinue(receiptCount)`.
3. `OnboardingView.painCardsPage` receives the int and stores in `@State private var receiptCount: Int = 0`.
4. `OnboardingPersonalSolutionView` accepts `let receiptCount: Int` and renders the receipt-aware line.

This is already wired in the current Codex impl per the Pain Cards spec. Verify on device that a `Caught me` count of 4 produces the expected Plan Reveal line.
