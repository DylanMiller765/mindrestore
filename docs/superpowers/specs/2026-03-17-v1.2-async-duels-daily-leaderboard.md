# v1.2 Design: Async Friend Challenges + Daily Challenge Leaderboard

## Overview

Two features for v1.2's "Social & Competitive" update:
1. **Async Friend Challenges** — challenge a friend to beat your score on any game via a shareable link
2. **Daily Challenge Leaderboard** — real Game Center leaderboard for daily challenge scores

## Feature 1: Async Friend Challenges

### Flow

1. **Player A** finishes any game → taps "Challenge a Friend" on results screen
2. App generates a deep link: `memori://challenge?game=reactionTime&seed=12345&score=288&name=Dylan`
3. Player A shares link via iOS share sheet (iMessage, WhatsApp, etc.)
4. **Player B** taps link → Memori opens → sees "Dylan challenged you to Reaction Time!"
5. Player B plays the same seeded game (identical content via SeededGenerator)
6. Player B finishes → sees side-by-side comparison (their score vs Dylan's)
7. Player B can share their result back or challenge someone else

### Architecture

**URL Scheme**: `memori://challenge`

**Query Parameters**:
- `game` — ExerciseType rawValue (e.g., "reactionTime", "colorMatch")
- `seed` — Int seed for SeededGenerator (ensures identical game content)
- `score` — Challenger's score (Int, game-specific: ms for reaction, count for math, etc.)
- `name` — Challenger's display name (URL-encoded)

**No backend needed** — all challenge data is encoded in the URL itself. This is a v1.2 approach; CloudKit can replace this in v1.3 for richer features.

### Components to Build/Modify

1. **ChallengeLink model** — struct to encode/decode challenge URLs
   - `static func create(game:seed:score:name:) -> URL`
   - `init?(url: URL)` — failable init from incoming URL

2. **DeepLinkRouter** — extend to handle `challenge` path
   - Parse query params into ChallengeLink
   - Set `pendingDestination = .challenge(ChallengeLink)`

3. **ChallengeAcceptView** — new view shown when opening a challenge link
   - Shows: "Dylan challenged you to Reaction Time!"
   - Shows challenger's avatar/name
   - "Accept Challenge" button → navigates to the game with the seeded content
   - "Not Now" dismissal

4. **ChallengeResultView** — shown after Player B finishes
   - Side-by-side score comparison (Player A vs Player B)
   - Win/lose/tie state with appropriate messaging
   - "Share Result" → generates TikTokDuelResultCard (already exists)
   - "Challenge Someone Else" → creates new challenge link with Player B's score

5. **Seed integration per game** — each game's ViewModel needs a `seed` parameter
   - When seed is provided, use SeededGenerator for all randomization
   - When no seed, use normal random (existing behavior)
   - Games: all 10 (Reaction Time, Color Match, Speed Match, Visual Memory, Number Memory, Math Speed, Dual N-Back, Chunking, Word Scramble, Memory Chain)

6. **Results screen update** — add "Challenge a Friend" button to every game's results view
   - Generates ChallengeLink with current game, seed, and score
   - Opens share sheet

### Seed Strategy Per Game

| Game | What Gets Seeded |
|------|-----------------|
| Reaction Time | Timing intervals between prompts |
| Color Match | Word/color mismatch sequence |
| Speed Match | Symbol sequence and match positions |
| Visual Memory | Which cells are highlighted per level |
| Number Memory | Digit sequences per level |
| Math Speed | Math problems (operands and operators) |
| Dual N-Back | Grid positions and audio letters |
| Chunking | Digit sequences to chunk |
| Word Scramble | Which words are chosen per round |
| Memory Chain | Grid item sequence order |

### Score Encoding

Each game submits a different score type. The URL encodes the raw game score (same value sent to Game Center):
- Reaction Time: average ms (lower = better)
- Color Match: composite (accuracy × 1000 + timeBonus)
- Speed Match: composite (accuracy × 1000 + timeBonus)
- Visual Memory: max level reached
- Number Memory: max digit span
- Math Speed: composite (correct × 1000 + timeBonus)
- Dual N-Back: max N level
- Chunking: correct digits
- Word Scramble: composite (words × 1000 + timeBonus)
- Memory Chain: longest chain

## Feature 2: Daily Challenge Leaderboard

### Design

Real Game Center leaderboard — no simulated/fake data. Shows actual players who completed today's daily challenge.

### Components

1. **Game Center Leaderboard** — already have `dailyChallengeScore` leaderboard ID registered
   - Submit score (0-1000) after daily challenge completion
   - Already exists in GameCenterService

2. **DailyChallengeView results update** — after completion, show:
   - Your score (0-1000)
   - Your rank: "You placed #X today" (fetched from Game Center)
   - "View Leaderboard" button → shows today's full leaderboard
   - If you're the only player: "You're #1! Share to challenge others."

3. **Daily Challenge Leaderboard View** — reuse existing LeaderboardView component
   - Filter to daily challenge category
   - Show real Game Center entries only
   - No simulated data — if 3 people played, show 3 people

### Score Submission

Daily challenge score is already calculated (0-1000). Just needs to be reported to Game Center after completion, which may already be wired up.

## What's NOT In Scope

- CloudKit storage for challenges (v1.3)
- Real-time 1v1 matchmaking (v1.3+, needs 1K+ users)
- Push notifications for challenges ("Dylan challenged you!") — requires backend
- Challenge history/pending challenges list — requires persistence
- Social friend system — uses share links instead

## Removing Simulated Leaderboard Data

All leaderboards should show real Game Center data only. Remove the mock score generation from GameCenterService. If Game Center has no entries, show an empty state: "No scores yet. Be the first!"

This applies to ALL leaderboards, not just daily challenge.
