# Memori v2.0 — Focus Mode

## Overview
Focus Mode lets users block distracting apps and replace that screen time with brain training. When a blocked app is opened, a shield appears prompting the user to complete a quick brain game to unlock it temporarily. This is Memori's key differentiator — no other app combines brain training with screen time blocking.

## Core User Flow

### Shield Interaction
1. User tries to open a blocked app (e.g., TikTok)
2. Shield appears with Memo mascot and two buttons:
   - "Play a game" → opens Memori via `memori://focus-unlock`
   - "Stay focused" → dismisses attempt, app stays blocked
3. Shield tone rotates based on daily attempt count:
   - Attempts 1-2: Friendly Memo (mascot-goal) — "Train your brain first!"
   - Attempts 3-4: Stats-driven — "3rd time today..."
   - Attempts 5+: Guilt trip Memo (mascot-low-score) — "You've already tried 5 times today"

### Quick Game
- Memori opens directly to quick game screen (no Home tab, no splash)
- Random game selected from the 10 available games
- Shortened versions (~20-30 seconds):
  - Reaction Time: 3 rounds
  - Color Match: 15 seconds
  - Number Memory: 2 rounds
  - Speed Match: 10 trials
  - Math Speed: 5 problems
  - Visual Memory: 3 rounds
  - Dual N-Back: 5 trials
  - Chunking: 2 rounds
  - Chimp Test: 3 rounds
  - Verbal Memory: 10 words
- Game completion is mandatory — force-quitting does NOT grant unlock

### After Game Completion
- Quick toast: "Unlocked for 15 min!" (2 seconds)
- ALL blocked apps unlock for the chosen duration (not just the one they tried to open)
- User manually switches to their desired app
- After duration expires, shields re-activate

### Disabling Focus Mode
- NOT available on the shield — only "Play a game" and "Stay focused"
- Available in Memori app → Focus Mode settings → "Turn off Focus Mode"
- 10-minute cooldown before it actually disables ("Focus Mode will turn off in 10 minutes")
- Prevents impulsive disabling while allowing genuine opt-out

## Focus Mode Setup Flow (4-5 screens)

1. **Intro** — Memo mascot + "Block distracting apps, train your brain instead"
2. **Pick apps** — FamilyActivityPicker. Free users see note: "Pick 1 app (unlock more with Ultra)". Ultra users: unlimited selection.
3. **Schedule** — "Always on" toggle or time/day picker for scheduled blocking

## Technical Architecture

### New Xcode Targets (must be added manually in Xcode)
1. **DeviceActivityMonitor extension** — monitors scheduled blocking intervals, fires when schedules start/end
2. **ShieldConfiguration extension** — provides custom shield UI with Memo mascot, rotating messages
3. **ShieldAction extension** — handles "Play a game" button tap, opens Memori via URL scheme

### Frameworks
- `FamilyControls` — AuthorizationCenter, FamilyActivityPicker
- `ManagedSettings` — ManagedSettingsStore for applying/removing shields
- `DeviceActivity` — DeviceActivityMonitor, DeviceActivitySchedule

### Shared State (App Group)
- Use existing App Group: `group.com.memori.shared`
- UserDefaults(suiteName:) for:
  - `focus_mode_enabled: Bool`
  - `focus_unlock_until: Date?` (when current unlock expires)
  - `focus_unlock_duration: Int` (minutes)
  - `focus_schedule_enabled: Bool`
  - `focus_schedule_start: Date`
  - `focus_schedule_end: Date`
  - `focus_daily_attempt_count: Int`
  - `focus_daily_attempt_date: Date`
  - `focus_cooldown_until: Date?` (for disable cooldown)

### URL Scheme
- `memori://focus-unlock` — triggers quick game flow
- App detects this URL in DeepLinkRouter, presents quick game immediately

### New Files
- `Services/FocusModeService.swift` — manages FamilyControls auth, ManagedSettingsStore, unlock/lock logic
- `Views/FocusMode/FocusModeSetupView.swift` — 4-step setup flow
- `Views/FocusMode/FocusModeCard.swift` — Home tab card
- `Views/FocusMode/FocusModeSettingsView.swift` — management screen
- `Views/FocusMode/QuickGameView.swift` — shortened game wrapper
- `Extensions/ShieldConfigurationExtension/` — shield UI extension
- `Extensions/ShieldActionExtension/` — shield button handler
- `Extensions/DeviceActivityMonitorExtension/` — schedule monitoring

## Edge Cases
- **Force-quit during quick game:** Unlock NOT granted. Shield stays up.
- **Already in Memori when shield URL fires:** Queue the quick game, show after current activity.
- **No internet:** Everything works offline — games are local, FamilyControls is on-device.
- **User has StretchCheck too:** Independent FamilyControls per app, no conflict.
- **Free user hits 1-app limit:** Show Ultra paywall when they try to add a second app.

## Analytics (PostHog)
- `focus_mode_enabled` / `focus_mode_disabled`
- `focus_shield_shown` (with attempt_count)
- `focus_unlock_game_started` (with game_type)
- `focus_unlock_game_completed` (with game_type, score)
- `focus_unlock_granted` (with duration_minutes)
- `focus_stayed_focused` (chose not to unlock)
- `focus_setup_completed` / `focus_setup_skipped` (in onboarding)
- `focus_cooldown_initiated` (user tried to disable)
