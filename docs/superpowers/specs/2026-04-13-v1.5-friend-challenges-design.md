# v1.5: Friend Challenges + Try Every Game

## Goal

Two features that work together: friend challenges create a viral loop that brings new users, and "try each game once" ensures those new users experience the full app before hitting the paywall.

## Feature 1: Friend Challenges

### Overview

After any game, players can challenge a friend by sharing a link. The friend plays the exact same game (same seed = same sequence/pattern), and sees a head-to-head comparison on the results screen. The friend can then "Challenge Back" to create a viral loop.

### User Flow — Sender

1. Player finishes any game → sees GameResultView
2. Taps **"Challenge a Friend"** button (new, below existing Share/Play Again)
3. iOS share sheet opens with text message + Vercel URL
4. Share message format: `"I got [display score] on [Game Name] [emoji] Think you can beat me? [vercel URL]"`
5. No image card — just text + link (share cards have near-zero usage per analytics)

### User Flow — Receiver (Has App)

1. Taps shared link → opens Vercel landing page
2. Page detects app installed → redirects to `memori://duel?game=X&seed=Y&score=Z&name=N`
3. App opens → DeepLinkRouter routes to ChallengeAcceptView (already built)
4. ChallengeAcceptView shows: challenger name, game name, "Accept Challenge" button
5. Taps Accept → launches game with same seed
6. After game → GameResultView shows normal results PLUS a challenger comparison banner at the top:
   - "Dylan got Level 8" with win/lose indicator
   - Green "You Win!" or red "They Win!" badge
7. "Challenge Back" button available → generates new challenge link with new seed

### User Flow — Receiver (No App)

1. Taps shared link → opens Vercel landing page
2. Page shows: "[Name] challenged you to [Game]!" with their score
3. "Download Memori" button → App Store link (https://apps.apple.com/app/id6760178716)
4. Challenge context is lost after download (acceptable for v1.5 — no deferred deep linking)

### Score Comparison

- **Winner determined by:** `leaderboardScore` (Int) — already standardized per game
- **Displayed as:** game-specific human-readable value (Level 7, 288ms, 92%, 9 digits, etc.)
- **Reaction Time special case:** lower is better (already handled in existing FriendChallengeResultView logic)

### ChallengeLink URL Format (Existing)

```
memori://duel?game=chimpTest&seed=34521&score=8&name=Dylan
```

Vercel equivalent:
```
memori-website-sooty.vercel.app/challenge?game=chimpTest&seed=34521&score=8&name=Dylan
```

### Vercel Landing Page (`/challenge`)

Single page at `memori-website-sooty.vercel.app/challenge` that:

1. Reads query params: `game`, `seed`, `score`, `name`
2. Displays: "[Name] challenged you to [Game Name]!" with their score
3. Primary CTA: Opens `memori://duel?...` URL (for users with app)

5. Simple, dark themed, matches app aesthetic
6. Static page — no backend, just client-side JS reading URL params

## Feature 2: Try Each Game Once

### Overview

New users can play each of the 10 games once before the 3/day free limit kicks in. This ensures they experience the full app before seeing the paywall, leading to better conversion.

### Logic

Current behavior: Free users are limited to 3 exercises per day. After 3, the paywall shows.

New behavior:
1. Track which game types the user has ever completed (persist in UserDefaults as a `Set<String>` of ExerciseType raw values)
2. When checking the daily limit, if the user is about to play a game type they've **never** played before, skip the limit check
3. Once they've tried every game at least once, the normal 3/day limit applies fully
4. Pro users are unaffected (no limits)

### Edge Cases

- User plays 3 games of types they've already tried → paywall as normal
- User plays 2 tried games + 1 untried game → all 3 allowed (2 counted toward limit, 1 is free trial)
- User has tried 9 of 10 games → the 10th game is always free regardless of daily count
- The "tried games" set persists forever (not reset daily)

### Implementation

- Add `triedGameTypes: Set<String>` to UserDefaults (not SwiftData — lightweight, doesn't need to sync)
- Modify `PaywallTriggerService.recordExerciseCompleted()` to check if game type is new
- If new: add to tried set, don't count toward daily limit
- If already tried: normal daily limit logic

### UI

- No special UI for this — it just works silently
- The daily limit pill on Home still shows the normal count (tried-once games don't consume a slot)
- Optional: on the Train tab, untried games could show a subtle "Try Free" badge (nice-to-have, not required)

## Out of Scope for v1.5

- Share card images for challenges (zero usage)
- Challenge history / win-loss tracking database
- Push notifications for received challenges
- Deferred deep linking (challenge context after fresh install)
- Universal Links (upgrade from Vercel redirect later)
- New games (Aim Trainer etc. — future update)
- Score decay / Memo sickness system
- Widgets

## Tech Stack

- **App:** SwiftUI, existing ChallengeLink/DeepLinkRouter/SeededGenerator infrastructure
- **Web:** Next.js page on existing Vercel deployment (memori-website-sooty)
- **No backend needed** — everything is client-side URL encoding
